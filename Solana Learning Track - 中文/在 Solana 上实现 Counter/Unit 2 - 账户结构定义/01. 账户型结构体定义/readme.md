# Content/**账户型结构体定义**

上一节中，我们已经导入了合约要使用的相关库，接下来，我们开始合约的编写。

首先，我们需要一个数据结构来存储我们的计数器信息。为了有效地存储计数信息，使用结构体（struct）来组织数据是一个很好的选择。

而为了便于管理和更新 ***Counter*** 的状态，我们会将 ***Counter*** 设计为一个独立的数据账户。这样做有几个重要的好处：

1. **数据持久化**：
    
    作为数据账户，***Counter*** 的状态将被持久化存储在区块链上。这意味着，无论程序何时被调用，***Counter*** 的信息都是最新且一致的。
    
2. **安全性和独立性**：
    
    将 ***Counter*** 作为独立账户，还能提高数据安全性和合约的健壮性，确保计数器状态不会因其他程序的操作而受到影响。
    

在 Anchor 框架中，我们使用`#[account]` 属性，它可以帮助我们将 ***Counter*** 结构体标注为数据账户。

**Syntax**

#[account] ，struct

- 提示
    
    ```rust
    #[account]
    pub struct Student{
        pub id: u64,
    }
    ```
